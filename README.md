# HCI (Human-Computer Interaction) E-Commerce UX Simulation

A dual-layered (Frontend + Backend) application specifically designed to test user behavior and susceptibility to *Dark Patterns* (manipulative design techniques) in an e-commerce environment.

## 🎯 Project Purpose

This application is **not** a regular e-commerce store. It is a simulated environment intended for Usability (UX) Testing. The core objective is to place participants in realistic shopping scenarios (login, add to cart, checkout) and measure how they react to deliberate UX flaws, hidden costs, fake social proofs, and "Roach Motel" patterns.

To effectively record these reactions, the system utilizes a **hidden Observer Mode** and a robust backend logging architecture that permanently stores the user's timestamps, button clicks, and error frequencies.

### ⚠️ Implemented Dark Patterns:
1. **Misdirection & Sneak into Basket:** Deceptive "Agree to Spam Newsletter" checkbox pre-selected during user registration.
2. **Hard-to-Satisfy Passwords:** Frustrating registration flow where password requirements are extremely strict on the backend but never explained on the UI.
3. **Hidden Fees & Sunk-Cost Fallacy:** Enormous "Service Fees" added only at the very final step of the credit card checkout, after the user has input all their shipping and payment details.
4. **Bait and Switch Navigation:** Clicking the "Electronics" navigation tab intentionally sends users to the "Sports" page.
5. **Fake Social Proof:** "Best seller" badges, fake 5-star ratings ("Expert Approved"), and countdown timers displaying "Only 3 items left! 47 people are viewing this right now!".
6. **Aggressive Pop-ups:** A screen-hijacking "Congratulations, 1000th Visitor!" popup with a massive accept button and a tiny, shaming reject text.
7. **Roach Motel:** A "Delete Account" button that traps the user in a confirmation loop and then throws a fake Server 500 Error, making it impossible to leave.

## 🛠️ Tech Stack & Architecture

This is a **Full-Stack Application** divided into two active components:

### 1. Backend (`/hci-backend`)
* **Node.js + Express.js**
* **JWT (JSON Web Token) + Bcrypt:** Secure authentication and password hashing.
* **JSON File Database (`db.json`):** A portable, serverless database that stores registered users, product states, active charts, and tracks the exact timestamps and actions (logs) generated by test subjects.

### 2. Frontend (`/hci-eticaret`)
* **React + Vite**
* **Context API:** Handles global state across the site, syncing guest local storage with the authenticated database.
* **Component-Based UI:** Highly modular setup with isolated `Layout`, `Shop`, `Modals`, `Auth`, and `ObserverPanel` logic.

## 🚀 How to Run the Application Locally

You will need to open **two separate terminal windows** to start both the server and the frontend client.

### Step 1: Start the Backend Server
1. Open your terminal.
2. Navigate to the backend directory:
   ```bash
   cd hci-backend
   ```
3. Install dependencies (only needed the first time):
   ```bash
   npm install
   ```
4. Start the server:
   ```bash
   node server.js
   ```
   *The backend will be running at `http://localhost:5000`.*

### Step 2: Start the Frontend Client
1. Open a new, second terminal window.
2. Navigate to the frontend directory:
   ```bash
   cd hci-eticaret
   ```
3. Install dependencies (only needed the first time):
   ```bash
   npm install
   ```
4. Start the Vite development server:
   ```bash
   npm run dev
   ```
   *The storefront will be live at `http://localhost:5173`.*

## 🔬 Observer Panel (Hidden Task Tracker)
When the React frontend is running, the researcher can open a hidden modal to create tracked scenarios for participants by using the keyboard shortcut:
**`CTRL + SHIFT + T`**

This panel allows you to:
- Assign predefined tasks to the human subject.
- Start a `Timer` that tracks precise task completion speeds (in seconds).
- View a live feed of the `Action Logs` (which buttons the user clicked, errors they made).
- Save the final `UX Report Data` securely into the database.
